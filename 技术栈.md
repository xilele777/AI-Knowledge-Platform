下面按你项目实际用到的技术栈整理，偏“面试可讲”的角度。

| 技术栈 | 在项目中的作用 | 项目中如何使用 | 面试常问 & 简明回答 |
|---|---|---|---|
| **Vue 3** | 前端主框架，负责页面、组件、响应式状态和交互 | [main.ts](f:/AI-Knowledge-Platform/src/main.ts) 通过 `createApp(App)` 创建应用；页面放在 `views/`，公共组件放在 `components/` | **问：Vue3 相比 Vue2 优势？** 答：Composition API 更适合复杂逻辑复用，响应式系统基于 Proxy，TypeScript 支持更好。 |
| **TypeScript** | 提供类型约束，降低 API 数据、组件 props、业务状态出错概率 | 类型集中在 [types](f:/AI-Knowledge-Platform/src/types)，如 `ChatMessage`、`Document`、`KnowledgeBase` | **问：TS 在项目中解决了什么问题？** 答：把后端表结构、API 入参出参、组件状态都显式建模，减少字段拼错和空值错误。 |
| **Vite** | 开发服务器和构建工具，提供快速启动、热更新、生产构建 | [vite.config.ts](f:/AI-Knowledge-Platform/vite.config.ts) 配置 Vue 插件和 `/api/llm` 代理 | **问：为什么用 Vite？** 答：开发阶段基于原生 ESM，启动和热更新快；生产阶段用 Rollup 打包。 |
| **Vue Router** | 管理页面路由、登录拦截、管理员权限控制 | [router/index.ts](f:/AI-Knowledge-Platform/src/router/index.ts) 定义 `/docs`、`/knowledge`、`/chat`、`/admin`，用 `beforeEach` 做鉴权 | **问：路由守卫做了什么？** 答：未登录跳 `/login`，已登录访问登录页跳业务页，非管理员访问 `/admin` 会被拦截。 |
| **Pinia** | 全局状态管理，保存用户状态和 AI 配置 | [stores/user.ts](f:/AI-Knowledge-Platform/src/stores/user.ts) 管理 `session/user/role`；[stores/aiConfig.ts](f:/AI-Knowledge-Platform/src/stores/aiConfig.ts) 管理用户 AI 配置 | **问：为什么用 Pinia？** 答：API 更简单，类型推导好，适合 Vue3；比 Vuex 更轻量。 |
| **Element Plus** | UI 组件库，提供表单、按钮、菜单、弹窗、布局等基础组件 | [main.ts](f:/AI-Knowledge-Platform/src/main.ts) 全局注册 `ElementPlus`；页面和布局大量使用 Element Plus 组件 | **问：如何做组件库样式定制？** 答：通过全局主题变量和覆盖样式文件，例如 [element-plus-overrides.css](f:/AI-Knowledge-Platform/src/styles/element-plus-overrides.css)。 |
| **Supabase** | 后端即服务，承担认证、数据库、RPC、数据存储 | [utils/supabase.ts](f:/AI-Knowledge-Platform/src/utils/supabase.ts) 初始化客户端；[api/auth.ts](f:/AI-Knowledge-Platform/src/api/auth.ts)、[api/documents.ts](f:/AI-Knowledge-Platform/src/api/documents.ts) 等封装业务访问 | **问：Supabase 在项目里替代了什么？** 答：替代传统后端的一部分能力，包括 Auth、Postgres 数据访问、RPC 和权限策略。 |
| **Supabase Auth** | 用户注册、登录、登出、恢复会话 | [api/auth.ts](f:/AI-Knowledge-Platform/src/api/auth.ts) 调 `signUp`、`signInWithPassword`、`getSession`、`onAuthStateChange` | **问：刷新页面如何保持登录？** 答：启动时 `userStore.initialize()` 调 `getSession()` 恢复会话，并绑定 Auth 状态监听。 |
| **Postgres / SQL 迁移** | 定义知识库、文档、聊天、统计等核心数据表 | SQL 脚本在 [supabase/sql](f:/AI-Knowledge-Platform/supabase/sql)，如 `knowledge_bases`、`documents`、`chat_messages` | **问：前端直连数据库安全吗？** 答：安全性依赖 Supabase RLS 策略，前端鉴权只能改善体验，不能替代数据库权限。 |
| **OpenAI 兼容 API** | 提供 AI 文本生成、流式回答、Embedding 能力 | [api/ai.ts](f:/AI-Knowledge-Platform/src/api/ai.ts) 请求 `/chat/completions`；[utils/vectorEmbedding.ts](f:/AI-Knowledge-Platform/src/utils/vectorEmbedding.ts) 请求 `/embeddings` | **问：为什么叫 OpenAI 兼容？** 答：只要服务商实现类似 `/chat/completions`、`/embeddings` 协议，就可以通过自定义 `baseUrl/model/apiKey` 接入。 |
| **SSE 流式响应** | AI 回答逐字/逐段返回，提升问答体验 | [api/ai.ts](f:/AI-Knowledge-Platform/src/api/ai.ts) 的 `generateAiTextStream()` 手动读取 `response.body.getReader()` 并解析 `data:` 事件 | **问：流式输出怎么实现？** 答：用 Fetch 读取 ReadableStream，维护 buffer，按 SSE 事件块解析增量内容。 |
| **RAG 知识库检索** | 把文档/文件切片后检索相关内容，再注入 Prompt 问答 | [utils/chunkText.ts](f:/AI-Knowledge-Platform/src/utils/chunkText.ts) 切片；[utils/retrieveChunks.ts](f:/AI-Knowledge-Platform/src/utils/retrieveChunks.ts) 检索；[utils/buildQaPrompt.ts](f:/AI-Knowledge-Platform/src/utils/buildQaPrompt.ts) 构建 Prompt | **问：RAG 流程是什么？** 答：资料切片入库，提问时召回相关片段，把片段和问题一起交给模型生成回答。 |
| **Embedding / 向量相似度** | 为语义检索做准备，通过向量计算文本相似度 | [utils/vectorEmbedding.ts](f:/AI-Knowledge-Platform/src/utils/vectorEmbedding.ts) 实现 `createEmbedding`、`cosineSimilarity`、`findTopSimilarChunks` | **问：余弦相似度有什么用？** 答：衡量两个向量方向是否接近，用于找出和问题语义最相关的知识片段。 |
| **md-editor-v3** | Markdown 文档编辑器 | 文档编辑页 [DocEditorView.vue](f:/AI-Knowledge-Platform/src/views/docs/DocEditorView.vue) 用它编辑 `content_md` | **问：为什么文档用 Markdown？** 答：结构清晰、便于切片、存储成本低，也适合后续作为知识库文本源。 |
| **ECharts / vue-echarts** | 管理后台统计图表展示 | 管理统计页 [AdminAnalyticsView.vue](f:/AI-Knowledge-Platform/src/views/admin/AdminAnalyticsView.vue) 展示登录趋势、AI 调用趋势等数据 | **问：图表数据从哪来？** 答：前端调用 [api/admin.ts](f:/AI-Knowledge-Platform/src/api/admin.ts) 的统计接口或 RPC，再传给 ECharts 配置。 |
| **localStorage** | 保存本地草稿，降低文档编辑丢失风险 | [utils/documentDraft.ts](f:/AI-Knowledge-Platform/src/utils/documentDraft.ts) 使用 `doc_draft_v1:{documentId}` 作为 key | **问：localStorage 有什么限制？** 答：只能本机本浏览器保存，不能多端同步，也不能存敏感信息。 |
| **CSS 主题与样式覆盖** | 统一 UI 风格，覆盖 Element Plus 默认样式 | [styles/theme.css](f:/AI-Knowledge-Platform/src/styles/theme.css) 放主题变量；[style.css](f:/AI-Knowledge-Platform/src/style.css) 放全局样式 | **问：如何维护统一风格？** 答：把颜色、间距、组件覆盖集中到主题样式文件，页面只做局部布局。 |

**面试主线可以这样讲：**  
这个项目基于 `Vue3 + TypeScript + Vite` 构建前端，用 `Pinia + Router` 管理状态和权限，用 `Supabase` 承担认证与数据存储，再通过 `OpenAI 兼容 API + SSE + RAG 检索` 实现知识库问答和 AI 写作能力。
